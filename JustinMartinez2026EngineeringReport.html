Glossary

Churn (line churn)
Total lines touched in a commit or period: insertions + deletions from Git (git log --numstat). It measures how much the diff moved, not net new code. Large churn often reflects lockfiles, generated bundles, vendor or SDK trees, or bulk moves - not necessarily hand-written feature work.
Density (impact index)
Commits x churn for a repo or bucket. A composite that rises when you commit both often and with large diffs. Use alongside narrative, review, and outcomes; it is not a productivity score.
Merge commits
Commits with more than one parent. Excluded here (--no-merges) so counts emphasize direct commits rather than branch integrations.
File rows / files touched
Count of numstat lines per commit (one row per path). Binary changes still count as one row.
Cobec vs personal bucket
Cobec: origin under jmartienzCobec or the cobec org. Personal: 10thony on GitHub. Personal breadth is useful context; calibrate with your manager how much OSS to surface.
Author filter
Commits matching the identities in the methodology table (emails and names used when generating the report).
High-churn commit
A single commit with churn >= 50,000 lines; often dependencies or bulk assets. Called out for separate interpretation with code review and business impact.
Active repositories
Repos under the scanned roots that had at least one matching commit in the date window.
